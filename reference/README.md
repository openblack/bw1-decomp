# reference/ — naming ground truth

## mac_symbols.tsv
**21,768 function names** extracted from the **un-stripped Mac build** of Black & White
(PowerPC PEF; every function carries its CodeWarrior-mangled name in an AIX traceback
table). This is the canonical naming source for the decomp: when a function, class, or
method needs a name or signature, check here first.

Format (tab-separated):

| column | meaning |
|---|---|
| `mac_addr` | `0x10000000 +` code-section offset in the Mac binary |
| `mangled` | CodeWarrior symbol, verbatim (ground truth) |
| `demangled` | best-effort readable form (96.1% fully parsed; falls back to mangled) |

Examples:
```
0x10254ab4  .__dt__18DecisionTreeAgendaFv          DecisionTreeAgenda::~DecisionTreeAgenda()
0x105b6634  .RemoveTown__11WorshipSiteFP4Town      WorshipSite::RemoveTown(Town*)
```

### Provenance & validation
Generated by `tools/mac_symbols.py` from the retail Mac binary (not shipped here);
tested by `tools/test_mac_symbols.py`:
- **completeness:** every literal `__ct__`/`__dt__` occurrence in the code section is
  captured by an extracted name (1497 + 1214, 0 missed)
- **tiling:** 0 overlaps across all 21,768 function intervals
- extraction is double-anchored: a name is accepted only if a traceback table's field
  chain lands exactly on its length field AND `tb_offset` points to a sane function start

Regenerate / re-verify:
```
python tools/mac_symbols.py "<path to Mac 'Black & White.bin'>" reference/mac_symbols.tsv
python tools/test_mac_symbols.py "<path to Mac 'Black & White.bin'>"
```

### Caveat
The `mac <addr>` numbers in older lift provenance comments came from a **different**
Mac build and do not equal these addresses (non-constant delta). **Cross-reference by
NAME, not by address.**
